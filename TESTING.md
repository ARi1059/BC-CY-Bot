# 端到端联调清单

部署后逐项验证；每完成一项打 ✅。所有用例假设 Bot Token、超级管理员 ID、Postgres 已正常启动。

> 准备：建议准备 3 个测试账号 + 1 个目标群 + 2 个频道（日志频道、出击报告频道）。

---

## 0. 启动验证

- [ ] `sudo systemctl start bccy-bot` 启动成功（`is-active` 返回 `active`）
- [ ] `sudo journalctl -u bccy-bot | grep super_admin_ensured` 看到 `action=created`（首次）或 `noop`（重启）
- [ ] 私聊 Bot 发 `/start` → 看到欢迎卡片
- [ ] 私聊 Bot 发 `/admin`（超管）→ 看到管理面板，其中 `[⚙️ 系统配置]` 按钮**可见**

## 1. 管理面板初始化

- [ ] `[👥 群组管理]` → `[➕ 添加群组]` → 转发目标群任一条消息 → 群组已添加
- [ ] `[🎓 邀请人管理]` → `[➕ 添加邀请人]` → 完成 6 步引导：
  - [ ] 输入"自审型测试老师"的 Telegram ID（用账号 A 的 ID）
  - [ ] 显示名："张老师"，组别："A组"
  - [ ] 选择刚添加的目标群组
  - [ ] 多选 3 项材料（约课记录 / 上课手势 / 出击报告）
  - [ ] 选择 `👤 自审型`
  - [ ] 确认创建 → 看到"邀请人 已创建"
- [ ] `[📡 日志频道]` → 绑定（转发日志频道任一条消息）
- [ ] `[📋 出击报告频道]` → 绑定

## 2. 自审型完整流（账号 A = 张老师，账号 B = 申请人）

**账号 B 侧（申请人）：**

- [ ] `/start` → 欢迎卡片 → `[🚀 开始申请入群]` → 看到邀请人列表（"张老师 · A组"）
- [ ] 点击张老师 → 看到 "(1/3) 请上传【约课记录】单张图片"
- [ ] 上传单张图片 → 看到 "✅ 已收到" + "(2/3) 请上传【上课手势】..."
- [ ] **测试媒体组拦截**：尝试一次性发 2 张图（点 + 多选）→ 应被拒绝："请单张提交"，且步骤不前进
- [ ] 上传单张上课手势图 → 进入 step 3
- [ ] 发送出击报告文本 → 进入预览
- [ ] `[✅ 确认提交]` → "申请已提交"

**账号 A 侧（张老师）：**

- [ ] 立即收到**两条消息**：① 媒体组（约课记录+上课手势）+ caption 是出击报告全文 ② 申请人信息 + `[✅ 通过][❌ 拒绝][👁 重发审核材料]`
- [ ] `[👁 重发审核材料]` → 再收到同样两条
- [ ] `[✅ 通过]`

**账号 B 侧：**

- [ ] 收到 "🎉 申请已通过" + `[🔗 点击加入群组]` 按钮 + **回群密钥**卡片（`BCCY-XXXX-XXXX-XXXX-XXXX`）
- [ ] **保存密钥**（后续测试要用）
- [ ] 点击链接 → 加入目标群

**频道侧：**

- [ ] 日志频道收到 `📥 新申请` 卡片
- [ ] 日志频道收到 `✅ 审核通过` 卡片（链接脱敏 t.me/+xxxx****）
- [ ] 日志频道收到 `🚪 链接已使用` 卡片（"✓ 一致"）
- [ ] 出击报告频道收到**转发的报告原文**（"Forwarded from 账号 B"），**没有图片，没有卡片**

## 4. 代审型完整流（多管理员并发）

**准备：**

- [ ] `/admin` → 管理员管理 → `[➕ 添加副管理员]` → 输入账号 D 的 Telegram ID
- [ ] `/admin` → 邀请人管理 → 添加一个 **代审型** 邀请人（review_mode = 代审型，挂名 inviter 可 /skip 不填 ID）

**账号 B 发起申请：**

- [ ] `/start` → 选代审型邀请人 → 走完 wizard 提交

**两位管理员（超级管理员 + 账号 D）：**

- [ ] **都**收到双消息（媒体组 + 按钮）
- [ ] 超管点 `[✅ 通过]`
- [ ] **同时**账号 D 看自己消息已被编辑为 `⏩ 已被 @xxx 处理`
- [ ] 账号 D 再点按钮 → 收到"该申请已被处理"提示
- [ ] DB 中 `applications.locked_by = 超管 ID`，`reviewed_by_type='admin'`

## 5. 拒绝流程

- [ ] 账号 B 再次申请
- [ ] 张老师收到双消息 → `[❌ 拒绝]` → 看到 `[✏️ 填写原因] [⏩ 跳过]`
- [ ] `[✏️ 填写原因]` → Bot 提示发送原因
- [ ] 发送 "材料不符" → 账号 B 收到 "❌ 您的申请未通过，原因：材料不符"
- [ ] 日志频道收到 `❌ 审核拒绝` 卡片（含原因）
- [ ] 张老师的审核消息编辑为 `❌ 已拒绝 by @xxx · 原因：材料不符`

## 6. 回群密钥（正常账号被踢路径）

**前置：** 账号 B 已通过审核进群，**已保存密钥**，目前还在群中。

- [ ] 账号 E（新账号，从未申请过）私聊 Bot 发 `/start` → `[🔑 我有回群密钥]`
- [ ] 粘贴账号 B 的密钥
- [ ] 收到 "✅ 验证通过 + 新链接 + 新密钥"
- [ ] 账号 E 点链接入群 → 进入目标群
- [ ] **账号 B 应被踢出群**（保留群 chat history，但成员列表无 B）
- [ ] 账号 B 仍可被重新邀请（kick 不是 ban，未来仍能加回）
- [ ] 日志频道收到 `🔑 回群密钥使用` + 清理动作 `✅ 已踢出原账号`
- [ ] DB 中旧 RecoveryKey.status='used'，新 RecoveryKey 链上 `previous_key_id` 指向旧 key

## 14. 报销系统（v2 §8.5）

### 14.1 初始化

- [ ] `/admin → [💰 报销管理] → [📋 系统配置]` 看到默认值（关闭/0/0/7/1）
- [ ] 超管点【▶️ 开启总开关】→ 状态变为已开启
- [ ] 设置固定金额 50 → 月预算 1000 → 重置日 1 → 冷却天数 7
- [ ] 点【♻️ 重置当前月余额至月预算】→ 月剩余变成 1000.00 元
- [ ] `[🎯 资格列表]` → 添加 1 个目标群 + 1 个频道（转发消息识别）

### 14.2 用户侧 4 层预校验

- [ ] 未通过入群审核的账号发 `/reimburse` → "您尚未通过入群审核"
- [ ] 关闭总开关时发 `/reimburse` → "报销功能当前未启用"
- [ ] 资格列表中删掉 1 个 → 再发 `/reimburse` → "您不符合报销资格"
- [ ] 完整通过预校验后进入 wizard 第 1 步

### 14.3 用户 wizard

- [ ] 第 1 步上传约课记录单图 → 进入第 2 步
- [ ] 第 2 步尝试发媒体组 → 被拒"请单张提交"
- [ ] 第 2 步上传上课手势 → 第 3 步
- [ ] 第 3 步发出击报告文本 → 进入预览
- [ ] 预览页【« 上一步】→ 回到第 3 步
- [ ] 预览页【✏️ 重新提交】→ 清空材料回到第 1 步
- [ ] 完整提交 → "✅ 报销申请已提交"

### 14.4 管理员审核

- [ ] 所有管理员私聊收到双消息（媒体组+caption 报告 / 申请人信息+按钮）
- [ ] 副管理员点【✅ 通过】→ 收到"✅ 已批准 #R... 请发送支付宝口令红包文本"
- [ ] 其他管理员的消息编辑为"⏩ 已被处理"
- [ ] 审核者发口令文本 → Bot 转发给申请人 + 审核消息变"💸 已付款"
- [ ] 月剩余预算扣减 50 元
- [ ] 日志频道收到 `💰 新报销 / ✅ 报销已批准 / 💸 报销已发放` 3 张卡片（口令未泄漏）

### 14.5 拒绝流程

- [ ] 再发起一份报销 → 任一管理员【❌ 拒绝】→ 二级菜单
- [ ] 【✏️ 填写原因】→ 发送原因 → 申请人收到带原因的拒绝消息
- [ ] 日志频道：`❌ 报销拒绝` 卡片含原因

### 14.6 预算不足自动退回

- [ ] 把月剩余手动改为 30（< 50） → 用户提交申请 → 管理员点通过 → 自动 reject，原因"本月预算已用尽，已退回"
- [ ] 日志频道收到 `⚠️ 报销被退回（预算不足）`

### 14.7 冷却时间

- [ ] 同一用户付款后 5 分钟内再发 `/reimburse` → "您还需等待 X 天才能再次申请"
- [ ] 给该用户添加单用户覆盖为 0 天（最小值 1）→ 验证立即可用（边界）

### 14.8 待付款补发

- [ ] 通过申请后 5 分钟内不发口令（或发 /cancel）→ 状态保留 approved
- [ ] `/admin → [💰 报销管理] → [💸 待付款]` 看到该条
- [ ] 点【💸 补发口令】→ Bot 提示等待口令 → 发文本 → 完成发放

### 14.9 历史记录

- [ ] `[📜 历史记录]` 看到最近 30 条，含 5 种状态图标
